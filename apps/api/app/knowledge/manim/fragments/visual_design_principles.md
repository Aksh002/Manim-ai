# Visual Design Principles

Use motion only when it explains structure, sequence, transformation, comparison, accumulation, or
temporal evolution. Static labeled diagrams are better when motion would distract.

Quality rules:

- Do not show everything at once; progressively disclose.
- Keep one or two focal animations active at a time.
- Use opacity hierarchy: primary elements full opacity, context around 0.35, grids/axes around 0.15.
- Assign colors semantic meaning and keep that meaning throughout the video.
- Keep edge text inside frame with comfortable margins.
- Long text must be shortened or constrained to the visible width.
- New equations need time to be read: slow reveal plus a pause.
- Aha moments deserve the longest pause.
- No more than about five or six important elements should compete for attention.
- Vary layout and animation type, but keep typography and palette cohesive.

Useful layout zones for 16:9:

- Top title/context zone.
- Center main visualization zone.
- Bottom formula or explanation zone.
- Side-by-side comparison zone for two perspectives.

Use generated plans to identify clutter, weak motion, poor contrast, missing waits, and unclear
visual hierarchy before writing code.
